A **Page** is one surface of the [Virtual Console](/virtual-console). The console
can have several pages — switch between them with the tabs in the toolbar —
letting you organise a big show into manageable screens (for example one page per
song, or one per area of the stage).

Unlike the other entries in this chapter, a page is not a widget you drag onto
the console: pages already exist as the tabs along the top. Its settings are
reached by entering **edit mode** with **no widget selected**, so that the right
panel shows the page's own properties.

## Settings

| Setting | What it does |
|---------|--------------|
| **Width** | The width of the page canvas, in pixels. Make it large enough for all the widgets you intend to place, or match a touchscreen's resolution. |
| **Height** | The height of the page canvas, in pixels. |
| <i class="fa fa-lock"></i> **Security → Set a PIN** | Protects the page with a PIN. When set, switching to the page asks for the PIN first, so sensitive controls can be locked away during a show. |

## Managing pages

| Button | What it does |
|--------|--------------|
| **Add page to the left** | Inserts a new, empty page immediately before the current one. |
| **Add page to the right** | Inserts a new, empty page immediately after the current one. |
| <i class="fa fa-trash" style="color:crimson"></i> **Delete this page** | Removes the current page and all the widgets on it (after asking you to confirm). |

## Switching and detaching

* **Left-click** a page tab in the toolbar to show that page.
* **Right-click** a page tab to **detach** it into its own window — useful for
  putting a page on a second monitor or touchscreen.

## Tips

* Give each page a meaningful name (edit its tab caption) so an operator can find
  the right screen quickly during a show.
* Use a **PIN** on pages that hold setup or master controls you don't want touched
  mid-show.
